# P1: Condition-Stratified Effect-Size Evaluation of Federated Acoustic Anomaly Detection

Reproducibility package for the manuscript:

> "Machine Type and Noise Outweigh Algorithm Choice: A Condition-Stratified
> Effect-Size Study of Federated Acoustic Anomaly Detection"
> (under preparation for IEEE Access, 2026).

This repository contains the exact training/analysis pipeline and the
aggregate result artifacts behind every number in the manuscript. It does
NOT redistribute the MIMII dataset (separate license; see Data Setup).

## Layout

```
pipeline/   training, sweep-generation, worker, and analysis code (as used)
results/    aggregate result artifacts cited in the manuscript
figures/    key sensitivity figures
```

Key result artifacts:

- `results/p1_results_aggregate.csv` — the audited 5,760-run main-sweep
  aggregate (FedAvg / FedProx mu=0.01 / Clustered FL / Personalized x
  4 machine types x 3 SNR x 6 Dirichlet alpha x 20 seeds).
- `results/main_stats/` — matched paired tests, factor ranges, additive and
  logit-AUROC effect sizes (partial eta-squared), seed ICC.
- `results/norm_ablation/` — 1,215-run normalization ablation paired deltas
  (central vs. federated sufficient-statistic vs. site-local).
- `results/e2_baseline_anchors/` — 360-run centralized-pooled / local-only
  training anchors.
- `results/e3_e4_reviewer_risk/` — 400-run FedProx-mu sensitivity and
  160-run LiteConvAE backbone sensitivity.
- `results/evidence_analysis/` — evidence-logged diagnostics (reconstruction
  error separability, site AUROC dispersion, Clustered-FL stability).

## Data Setup

Download MIMII (Purohit et al., DCASE 2019) from
https://zenodo.org/record/3384388 and arrange as:

```
<data_root>/{-6dB,0dB,6dB}/{fan,pump,slider,valve}/...
```

Set `data_root` in the run config JSON (default `~/abada-night/data/mimii`;
any path works).

## Reproducing a Sweep

```bash
# 1. generate run configs (writes one JSON per run)
python pipeline/gen_backlog.py --out <backlog_dir>

# 2. start one worker per GPU; workers atomically claim configs
python pipeline/p1_worker.py   # see start_p1_workers.sh for the 8-GPU launcher

# 3. aggregate + statistics
python pipeline/analyze_p1_main_stats.py
```

Reviewer-risk profiles (E2 anchors, E3 FedProx-mu, E4 backbone) are
generated by `pipeline/gen_reviewer_risk_backlog.py`; the normalization
ablation by `pipeline/gen_normalization_ablation_backlog.py`; evidence
logging is enabled per-config (`log_evidence`) and analyzed by
`pipeline/analyze_p1_evidence.py`.

Requirements: Python 3.11+, PyTorch (CUDA), torchaudio or librosa, numpy,
pandas, scipy. `wandb` is optional (`use_wandb: false` disables it).
Carbon-annotation hooks (`pipeline/carbon.py`) require an Electricity Maps
token via `ELECTRICITYMAPS_TOKEN` and are not needed to reproduce the paper.

## Notes

- Path defaults reflect the original cluster layout (`~/abada-night/...`);
  every path is overridable via config fields or CLI flags.
- The federated sites are simulated via Dirichlet partitioning of MIMII
  normal examples; see the manuscript's Method and Limitations sections for
  the exact claim boundary.
